вқ‘It is not a great idea to process an entire image because many parts in an image
may not contain any useful information. Therefore, by segmenting the image, we

SCALE INVARIANT FEATURE
TRANSFORM (SIFT)
вҖў The SIFT (Scale-Invariant Feature Transform) algorithm is a computer vision
technique used for feature detection and description.
вҖў SIFT stands for Scale-Invariant Feature Transform and was first presented in
2004, by D.Lowe, University of British Columbia. SIFT is invariance to image
scale and rotation
вҖў It detects distinctive key points or features in an image that are robust to
changes in scale, rotation, and affine transformations.
вҖў SIFT(scale invariant feature transform) works by identifying key points
based on their local intensity extrema and computing descriptors that
capture the local image information around those key points.
вҖў These descriptors can then be used for tasks like image matching, object
recognition, and image retrieval.
ADVANTAGE OF SIFT
вҖў Scale and Rotation Invariance: SIFT features are robust to changes in scale and rotation, making them suitable for a wide range
of applications.
вҖў Distinctive Descriptors: The 128-dimensional descriptors are highly distinctive, allowing for accurate matching of features
between images.
вҖў Robustness to Noise and Illumination Changes: SIFT features are relatively insensitive to noise and changes in illumination,
enhancing their reliability.
вҖў 3D viewpoint invariance: the feature remains the same even with the change in 3D viewpoint
вҖў Occlusion Invariance : Handles partial occlusions of objects within the scene
APPLICATIONS OF SIFT
1. Object Recognition: SIFT is widely used in object recognition tasks, where the goal is to identify objects in images regardless of their
orientation, scale, or viewpoint. By matching SIFT features between a target image and reference images, objects can be reliably identified.
2. Image Stitching: In panoramic photography and image stitching, SIFT is employed to find corresponding points between overlapping
images. These matching points are then used to align and blend the images seamlessly, creating a single, wide-angle view.
3. 3D Reconstruction: SIFT is used in 3D reconstruction to identify matching points between images taken from different angles. These
matches are used to triangulate the positions of points in 3D space, enabling the reconstruction of the scene's 3D structure.
4. Robot Navigation: SIFT can be used for navigation and mapping by autonomous robots. By detecting and matching features in the
environment, robots can localize themselves and build maps of their surroundings.
5. Image Retrieval: It involves finding images similar to a given query from a large database

INTERSECTION OVER UNION (IOU)
вҖў Intersection over Union (IoU) is used to evaluate the performance of
object detection by comparing the ground truth bounding box to the
predicted bounding box
вҖў To apply Intersection over Union to evaluate an (arbitrary)
object detector we need:
вһў The ground-truth bounding boxes (i.e., the hand-
labeled bounding boxes from the testing set that
specify where in the image our object is).
вһў The predicted bounding boxes from our model.
In the numerator, we compute the area of
overlap between the predicted bounding box and
the ground-truth bounding box.
The denominator is the area of union, or more simply, the
area encompassed by both the predicted bounding box
and the ground-truth bounding box.
Dividing the area of overlap by the area of union yields our
final score вҖ” the Intersection over Union.
INTERSECTION OVER UNION (IOU)
вҖў The Intersection over Union (IoU) equation measures the overlap between
two boundaries, such as a predicted bounding box and a ground-truth box. It
is a fundamental metric in computer vision for evaluating object detection and
image segmentation models.

PROPERTIES OF IOU
IoU has several important properties that make it a robust metric for
evaluating object detection performance:
вҖў Non-negativity: IoU values are always greater than or equal to zero
вҖў Symmetry: The value of IoU is the same regardless of the order in which the
two shapes are compared
вҖў Scale Insensitivity: IoU is invariant to the scale of the shapes being
compared, making it a versatile metric
IOU LIMITATIONS
IoU is essential for assessing the accuracy of object detection models, as it
quantifies how well the predicted bounding boxes match the ground truth
boxes. However, it does have limitations, such as providing no information
about the distance between non-overlapping boxes. Various extensions and
alternatives to IoU, like GIoU, DIoU, CIoU, and others, have been developed to
address these limitations and improve the performance of object detection
algorithms.

THE PROBLEM WITH IOU LOSS
вҖў No Gradient for Non-Overlapping Boxes: The biggest issue with IoU loss is that if
the predicted bounding box and the ground truth box donвҖҷt overlap at all, the IoU is
zero. This means the gradient of the loss is also zero, providing no information to
the model about how to adjust the predicted box to move closer to the ground
truth. This is a problem, especially during early training when predictions are often
far off.
вҖў DoesnвҖҷt Distinguish Between Close and Far: IoU only measures the area of overlap.
It doesnвҖҷt tell you anything about how close the boxes are if they donвҖҷt overlap. A
prediction thatвҖҷs slightly off and a prediction thatвҖҷs very far away both result in an
IoU of zero, even though the slightly off prediction is clearly better.

NON-MAX SUPPRESSION
вҖў Non Maximum Suppression is a computer
vision method that selects a single entity out
of many overlapping entities (for example
bounding boxes in object detection). The
criteria is usually discard entities that are
below a given probability bound
вҖў Non-maximum suppression (NMS) is a post-
processing technique used in object
detection to eliminate duplicate detections
and select the most relevant detected
objects. This helps reduce false positives
and the computational complexity of a
detection algorithm.
NMS ALGORITHM
1. .Discard all bounding boxes with Pc<0.6 (low probability to be an object)
2. .Select one box with the highest Pc
3. Calculate IoU (DIoU) between selected box and all other boxes
4. Discard Boxes with IoU >0.5 (Highly overlapped with selected box)
5. Repeat 2,3,4 excluding the previously selected predicted box
6. .Stop when there is no box to discard
NMS ALGORITHM
вҖў The idea behind NMS is straightforward.
It works by comparing the confidence
scores of the proposed bounding boxes
and eliminating the ones that overlap
significantly with a higher-scoring
bounding box.
вҖў NMS helps to reduce the number of false
positives and improves the overall detection
accuracy of the model. Without NMS, the
model may generate multiple bounding box
predictions for the same object, leading to
duplication in the output and a decrease in
accuracy.

вқ‘ This approach utilizes a selective search algorithm to generate approximately 2,000 region proposals, which are then

HOW R-CNN WORK
R-CNN follows a two-stage approach:
1. Region Proposal (Where are the objects?)
вҖў It first proposes around 2000 regions in the image that might contain objects.
вҖў These are called Region Proposals and are generated using a method like Selective Search.
2. Classification + Bounding Box Regression
вҖў For each region:
вҖў Crop and resize it to a fixed size (e.g., 224x224).
вҖў Feed it into a CNN (like AlexNet) to extract features.
вҖў Use those features to:
вҖў Classify what object (if any) is in that region.
вҖў Refine the bounding box to better fit the object using regression.
REGION PROPOSALS
вқ‘ R-CNNs begin by generating region proposals, which are smaller sections of the
image that may contain the objects we are searching for.
вқ‘ The algorithm employs a method called selective search, a greedy approach that
generates approximately 2,000 region proposals per image. Selective search
effectively balances the number of proposals while maintaining high object recall,
ensuring efficient object detection.
вқ‘ By limiting the number of regions for detailed analysis, this method enhances the
overall performance of the R-CNN in detecting objects within images.
вқ‘ A significant drawback of R-CNN is that it is computationally expensive, requiring the CNN
to be run on each ROI individually.

REGION PROPOSAL NETWORK (RPN)
вҖў RPN is a fully convolutional network that
predicts the object bounds by learning from
feature maps extracted from a base network.
It has a classifier that returns the probability of
the region and a regressor that returns the
coordinates of bounding boxes.
вҖў It acting as the core component of the Faster R-
CNN architecture, significantly reducing
detection time compared to earlier methods.
How it Works (The Mechanics)
The RPN operates on a "Feature Map" (a condensed, numerical version of your image created
by a backbone like VGG or ResNet).
1.Sliding Window: It slides a small spatial window(region) over the feature map.
2.Anchor Boxes: At each position, it evaluates several "anchors"вҖ”predefined boxes of
different scales (small, medium, large) and aspect ratios (tall, wide, square).
3.Two-Head Output: For every single anchor box, the RPN calculates two things:
вҖўObjectness Score: A probability of whether the box contains an actual object versus just
being background.
вҖўBoundary Refinement: Mathematical offsets (regressions) to tweak the box coordinates
so they fit the object better.
4.Proposal Generation: It keeps the boxes with the highest scores and passes them to the
next stage of the network for final classification (e.g., "Is this a car or a person?").

3.Anchor Initialization: At the center of every grid cell (the Anchor Point), generate $K$
